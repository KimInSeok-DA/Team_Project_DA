# OULAD 데이터 사전 (팀 참고용)

영국 Open University 데이터라서 컬럼명이 영어고, 학력·지역 구분도 영국 기준이라 헷갈리기 쉬움.
작업하다 컬럼 뜻이 헷갈리면 이 문서부터 확인.

## 1. 원본 컬럼 (studentInfo 등)

### 인적 정보

| 컬럼명 | 뜻 | 값 종류 및 설명 |
|---|---|---|
| `gender` | 성별 | `M`=남성, `F`=여성 |
| `disability` | 장애 등록 여부 | `Y`=있음, `N`=없음 |
| `age_band` | 연령대 | `0-35`(35세 미만), `35-55`(35~55세), `55<=`(55세 이상) |
| `region` | 거주 지역 | 잉글랜드 각 지역 + 스코틀랜드/웨일스/아일랜드 등 영국 내 지역명. 국적이 아니라 "학생이 등록 당시 살던 행정구역" |

### 학력·경제 수준 (제일 헷갈리는 부분)

| 컬럼명 | 뜻 | 값 종류 및 설명 |
|---|---|---|
| `highest_education` | 입학 시점 최종 학력 | 영국 학제 기준, 한국 학제와 1:1로 안 맞음. 아래 표 참고 |
| `imd_band` | 거주 지역의 빈곤 지수(백분위 구간) | **숫자가 낮을수록 빈곤한 지역**. `0-10%`=가장 빈곤한 지역, `90-100%`=가장 부유한(박탈 적은) 지역. 학생 개인 소득이 아니라 "그 학생이 사는 동네"의 지표임 |
| `studied_credits` | 이번 학기에 수강 신청한 총 학점 수 | 숫자가 클수록 더 많은 과목/부담을 신청한 것 |
| `num_of_prev_attempts` | 이 과목을 이전에 재수강한 횟수 | `0`=처음 수강, `1`이상=재수강 |

**`highest_education` 값 상세 (낮은 학력 → 높은 학력 순):**

| 값 | 의미 | 한국 기준 대략 비교 |
|---|---|---|
| `No Formal quals` | 공식 자격 없음 | 정규 학력 인증 없음 |
| `Lower Than A Level` | 중등교육(고교) 과정 미수료 | 고졸 미만 |
| `A Level or Equivalent` | 중등교육 졸업 자격(대입 자격시험 수준) | 고졸(수능 응시 자격) 정도 |
| `HE Qualification` | 대학/고등교육기관에서 1년 이상 정규과정 이수 | 전문대 수료~학사 정도 |
| `Post Graduate Qualification` | 석사 학위 수준 이상 | 대학원(석사 이상) |

### 과목/학기 구분

| 컬럼명 | 뜻 | 값 종류 및 설명 |
|---|---|---|
| `code_module` | 과목 코드 | 실제 과목명 대신 `AAA`~`GGG` 7개로 익명화됨. 각 코드가 실제로 어떤 과목인지는 공개 안 됨 |
| `code_presentation` | 개설 학기 | `연도` + `B`/`J`. `B`=2월 개강 학기, `J`=10월 개강 학기 (예: `2013J`=2013년 10월 개강) |
| `module_presentation_length` | 그 학기의 총 수업 일수 | 보통 240~270일 사이 |

### 최종 결과

| 컬럼명 | 뜻 | 값 종류 및 설명 |
|---|---|---|
| `final_result` | 과목 최종 성적 결과 | `Distinction`=우수 통과, `Pass`=통과, `Fail`=낙제(수강은 완료), `Withdrawn`=중도 이탈(포기) |

### 날짜/시간 관련 (모두 "개강일(0일) 기준 며칠째"인 상대값, 실제 달력 날짜 아님)

| 컬럼명 | 뜻 |
|---|---|
| `date_registration` | 수강 등록일 (음수면 개강 전에 미리 등록한 것) |
| `date_unregistration` | 수강 취소(이탈)일. 값이 없으면 이탈 안 하고 끝까지 수강한 것 |
| `date_submitted` (studentAssessment) | 과제/시험 제출일 |
| `date` (assessments) | 과제/시험 마감일 |
| `date` (studentVle) | 클릭 로그가 발생한 날짜 |

## 2. 우리가 SQL로 만든 분석용 컬럼 (`v_student_early_signal` / `통합쿼리.csv`)

| 컬럼명 | 뜻 |
|---|---|
| `label_churn` | **최종적으로** 이탈했는지 여부 (`final_result = Withdrawn`이면 True) |
| `label_churn_28d` | **개강 후 28일 이내에** 이탈했는지 여부 — 이번 프로젝트의 핵심 예측 대상(타겟) |
| `n_assessment_submitted_28` | 개강 후 28일 안에 제출한 과제/시험 개수 |
| `avg_score_28` | 그 28일 안에 제출한 과제들의 평균 점수 (제출한 게 없으면 값 없음) |
| `avg_submit_delay_28` | 마감일 대비 평균 제출 시점. 음수=마감 전에 미리 제출, 양수=마감 늦게 제출 |
| `total_click_28` | 28일 동안 온라인 학습 시스템(VLE) 총 클릭 수 |
| `active_days_28` | 28일 중 실제로 접속해서 활동한 날 수 |
| `distinct_resources_28` | 28일 동안 열람한 서로 다른 학습자료 개수 |

## 3. 모델링할 때 절대 피처로 넣으면 안 되는 컬럼 (정답 유출)

`final_result`, `label_churn`, `date_unregistration` — 이 셋은 타겟(`label_churn_28d`)을 계산하는 데 쓰인 재료라서, 예측 모델의 입력값으로 넣으면 안 됨 (모델이 답을 미리 보고 맞히는 셈이 됨).
