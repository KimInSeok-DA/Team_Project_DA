# OULAD 데이터 사전 (팀 참고용) — CSV 파일별 정리

영국 Open University 데이터라서 컬럼명이 영어고, 학력·지역 구분도 영국 기준이라 헷갈리기 쉬움.
작업하다 컬럼 뜻이 헷갈리면 이 문서부터 확인. **원본 CSV 7개 + 우리가 만든 분석용 파일 1개,
총 8개 파일을 파일 단위로 정리**했습니다.

## 목차

| # | 파일명 | 한 줄 설명 |
|---|---|---|
| 1 | `studentInfo.csv` | 학생 인적사항 + 최종 성적(타겟 변수의 원재료) |
| 2 | `studentRegistration.csv` | 학생별 수강 등록일/취소일 |
| 3 | `courses.csv` | 과목-학기 메타데이터(수업 일수) |
| 4 | `assessments.csv` | 과제/시험 "계획서"(마감일, 배점, 유형) |
| 5 | `studentAssessment.csv` | 학생별 과제 제출 "실적"(제출일, 점수) |
| 6 | `vle.csv` | 온라인 학습자료 "목록"(자료 유형) |
| 7 | `studentVle.csv` | 학생별 일자별 클릭 로그 |
| 8 | 통합쿼리.csv (`v_student_early_signal`) | 우리가 SQL로 만든 분석용 파일 |
| - | 모델링 주의사항 | 정답 유출(피처로 넣으면 안 되는 컬럼) |

공통으로 알아둘 것: `code_module`(과목 코드) + `code_presentation`(개설 학기)은 거의 모든 파일에
공통으로 들어있는 **조인 키**입니다. 여기에 `id_student`(학생 고유 ID)까지 합치면 "어떤 학생이
어떤 과목/학기를 들었는지"를 유일하게 식별할 수 있습니다.

---

## 1. `studentInfo.csv` — 학생 인적사항 + 최종 성적

학생 1명이 수강한 과목-학기마다 1행. 인구통계 정보와 최종 결과(타겟 변수의 원재료)가 여기 다 있음.

### 식별자 / 과목·학기 구분

| 컬럼명 | 뜻 | 값 종류 및 설명 |
|---|---|---|
| `id_student` | 학생 고유 ID | 다른 테이블과 조인할 때 쓰는 마스터 키 |
| `code_module` | 과목 코드 | 실제 과목명 대신 `AAA`~`GGG` 7개로 익명화됨. 각 코드가 실제로 어떤 과목인지는 공개 안 됨 |
| `code_presentation` | 개설 학기 | `연도` + `B`/`J`. `B`=2월 개강 학기, `J`=10월 개강 학기 (예: `2013J`=2013년 10월 개강) |

### 인적 정보

| 컬럼명 | 뜻 | 값 종류 및 설명 |
|---|---|---|
| `gender` | 성별 | `M`=남성, `F`=여성 |
| `disability` | 장애 등록 여부 | `Y`=있음, `N`=없음 |
| `age_band` | 연령대 | `0-35`(35세 미만), `35-55`(35~55세), `55<=`(55세 이상) |
| `region` | 거주 지역 | 잉글랜드 각 지역 + 스코틀랜드/웨일스/아일랜드 등 영국 내 지역명. 국적이 아니라 "학생이 등록 당시 살던 행정구역" |

### 학력·경제 수준 (제일 헷갈리는 부분)

| 컬럼명 | 뜻 | 값 종류 및 설명 |
|---|---|---|
| `highest_education` | 입학 시점 최종 학력 | 영국 학제 기준, 한국 학제와 1:1로 안 맞음. 아래 표 참고 |
| `imd_band` | 거주 지역의 빈곤 지수(백분위 구간) | **숫자가 낮을수록 빈곤한 지역**. `0-10%`=가장 빈곤한 지역, `90-100%`=가장 부유한(박탈 적은) 지역. 학생 개인 소득이 아니라 "그 학생이 사는 동네"의 지표임. 결측이 있을 수 있음(비율 확인 필요) |
| `studied_credits` | 이번 학기에 수강 신청한 총 학점 수 | 숫자가 클수록 더 많은 과목/부담을 신청한 것 |
| `num_of_prev_attempts` | 이 과목을 이전에 재수강한 횟수 | `0`=처음 수강, `1`이상=재수강 |

**`highest_education` 값 상세 (낮은 학력 → 높은 학력 순):**

| 값 | 의미 | 한국 기준 대략 비교 |
|---|---|---|
| `No Formal quals` | 공식 자격 없음 | 정규 학력 인증 없음 |
| `Lower Than A Level` | 중등교육(고교) 과정 미수료 | 고졸 미만 |
| `A Level or Equivalent` | 중등교육 졸업 자격(대입 자격시험 수준) | 고졸(수능 응시 자격) 정도 |
| `HE Qualification` | 대학/고등교육기관에서 1년 이상 정규과정 이수 | 전문대 수료~학사 정도 |
| `Post Graduate Qualification` | 석사 학위 수준 이상 | 대학원(석사 이상) |

### 최종 결과

| 컬럼명 | 뜻 | 값 종류 및 설명 |
|---|---|---|
| `final_result` | 과목 최종 성적 결과 | `Distinction`=우수 통과, `Pass`=통과, `Fail`=낙제(수강은 완료), `Withdrawn`=중도 이탈(포기). **타겟 변수(`label_churn`)의 원재료** |

## 2. `studentRegistration.csv` — 등록일/취소일

학생별 수강 등록·취소 "날짜" 기록. 모든 날짜는 **개강일(0일) 기준 며칠째인 상대값**이며, 실제
달력 날짜가 아닙니다.

| 컬럼명 | 뜻 | 값 종류 및 설명 |
|---|---|---|
| `id_student` / `code_module` / `code_presentation` | 조인 키 | studentInfo와 동일 |
| `date_registration` | 수강 등록일 | 음수면 개강 전에 미리 등록한 것 |
| `date_unregistration` | 수강 취소(이탈)일 | 값이 없으면(NaN) 이탈 안 하고 끝까지 수강한 것. `final_result="Withdrawn"`과 100% 일치하지는 않으니 주의(3번 확인 참고) |

**주의**: `final_result == "Withdrawn"`(studentInfo)와 `date_unregistration`이 있는 학생
(studentRegistration)이 정확히 같은 집합은 아닙니다. 소수의 불일치 사례가 있으니, "이탈 여부"는
`final_result`/`label_churn` 기준으로, "이탈 시점"이 필요할 때만 `date_unregistration`을 보조로
쓰는 것이 안전합니다.

## 3. `courses.csv` — 과목-학기 메타데이터

과목-학기 조합당 1행(22개)뿐인 작은 파일. 학생 정보는 없고, 그 학기가 며칠짜리였는지만 담고
있습니다.

| 컬럼명 | 뜻 | 값 종류 및 설명 |
|---|---|---|
| `code_module` / `code_presentation` | 조인 키 | studentInfo와 동일 |
| `module_presentation_length` | 그 과목-학기의 총 수업 일수 | 보통 234~269일 사이 |

## 4. `assessments.csv` — 과제/시험 "계획서"

교수가 짜놓은 과제 스케줄. 학생별 제출 기록이 아니라, **이 과목-학기에 어떤 과제가 언제
마감이고 배점이 몇 %인지**를 정의한 파일입니다.

| 컬럼명 | 뜻 | 값 종류 및 설명 |
|---|---|---|
| `id_assessment` | 과제 고유 ID | studentAssessment.csv와 조인할 때 쓰는 키 |
| `code_module` / `code_presentation` | 조인 키 | studentInfo와 동일 |
| `assessment_type` | 과제 유형 | `TMA`=튜터가 채점하는 리포트, `CMA`=컴퓨터 자동채점 퀴즈, `Exam`=기말시험 |
| `date` | 과제 마감일 | 개강일(0일) 기준 며칠째. **studentVle의 `date`나 studentAssessment의 `date_submitted`와 같은 상대 기준을 씀** |
| `weight` | 성적 반영 비율(%) | `0`이면 성적에 안 들어가는 연습용 문제. 이탈 분석에는 `weight > 0`인 과제만 써야 함 |

**주의**: `Exam` 유형은 대부분 studentAssessment.csv에 제출 기록이 없습니다(별도 오프라인/지필
시험이라 이 데이터셋에 안 잡히는 것으로 추정). 과제 제출 패턴 분석은 사실상 TMA/CMA 중심입니다.

---

## 5. `studentAssessment.csv` — 학생별 과제 제출 "실적"

assessments.csv(계획서)와 짝을 이루는 "실제로 학생이 낸 결과" 파일.

| 컬럼명 | 뜻 | 값 종류 및 설명 |
|---|---|---|
| `id_assessment` | 어떤 과제인지 | assessments.csv와 조인 |
| `id_student` | 어떤 학생인지 | studentInfo와 조인 |
| `date_submitted` | 과제/시험 제출일 | 개강일 기준 상대값. 마감일(`assessments.date`)보다 크면 지연 제출 |
| `is_banked` | 점수 이월 여부 | `1`=예전에 통과한 과목을 재수강하면서 과거 점수를 그대로 인정받아 이월. **이 경우 `date_submitted`는 실제 제출일이 아니라 의미 없는 고정값(-1)이므로, 제출 시점을 분석할 땐 반드시 `is_banked=0`만 걸러서 써야 함** |
| `score` | 과제 점수 | 0~100점. 극소수 결측 있음(주로 채점 미완료) |

## 6. `vle.csv` — 온라인 학습자료 "목록"

학생 활동 기록이 아니라, **그 과목-학기에 어떤 학습자료가 있고 어떤 종류인지**를 정의한 카탈로그
파일입니다. studentVle.csv와 `id_site`로 조인해서 "학생이 어떤 종류의 자료를 눌렀는지" 알아낼 때
씁니다.

| 컬럼명 | 뜻 | 값 종류 및 설명 |
|---|---|---|
| `id_site` | 자료 고유 ID | studentVle.csv와 조인하는 키 |
| `code_module` / `code_presentation` | 조인 키 | studentInfo와 동일 |
| `activity_type` | 자료 유형 | 주요 값: `forumng`(토론게시판), `quiz`(퀴즈), `homepage`(메인화면), `oucontent`(학습 콘텐츠), `resource`(자료실), `subpage`(보조페이지), `url`, `glossary`, `oucollaborate`, `ouelluminate`(화상강의), `page`, `dataplus`, `dualpane`, `folder`, `sharedsubpage`, `questionnaire`, `externalquiz`, `htmlactivity`, `repeatactivity` 등 |
| `week_from` / `week_to` | 권장 학습 시작/종료 주차 | **결측이 매우 많음(약 82%)**. forumng·homepage처럼 학기 내내 상시로 열려있는 자료는 애초에 "권장 주차" 개념이 없어서 결측인 것으로, 오류가 아니라 정상임 |

---

## 7. `studentVle.csv` — 학생별 일자별 클릭 로그

이 데이터셋에서 가장 큰 파일(약 1,065만 행). "학생이 어떤 자료를 그 날 몇 번 클릭했는지" 기록.

| 컬럼명 | 뜻 | 값 종류 및 설명 |
|---|---|---|
| `id_student` / `code_module` / `code_presentation` | 조인 키 | studentInfo와 동일 |
| `id_site` | 클릭한 자료 ID | vle.csv와 조인해서 자료 유형(`activity_type`) 확인 |
| `date` | 활동일 | 개강일(0일) 기준 며칠째. 음수면 개강 전 사전 접속 |
| `sum_click` | 그날 그 자료를 클릭한 횟수 | **주의: (학생,자료,날짜) 조합이 한 행이 아닐 수 있음(같은 날 여러 세션이 별도 행으로 기록되는 경우가 약 35%).** "하루 총 클릭수"를 구할 땐 행 개수를 세지 말고 반드시 `sum_click`을 합산(sum)해야 함 |

## 8. 통합쿼리.csv (`v_student_early_signal`) — 우리가 SQL로 만든 분석용 파일

위 7개 원본 파일을 조합해서 만든 **파생 컬럼 전용** 파일. 원본에는 없는, 우리가 직접 계산한
값들입니다.

| 컬럼명 | 뜻 |
|---|---|
| `label_churn` | **최종적으로** 이탈했는지 여부 (`final_result = Withdrawn`이면 True) |
| `label_churn_28d` | **개강 후 28일 이내에** 이탈했는지 여부 — 이번 프로젝트의 핵심 예측 대상(타겟) |
| `n_assessment_submitted_28` | 개강 후 28일 안에 제출한 과제/시험 개수 |
| `avg_score_28` | 그 28일 안에 제출한 과제들의 평균 점수 (제출한 게 없으면 값 없음) |
| `avg_submit_delay_28` | 마감일 대비 평균 제출 시점. 음수=마감 전에 미리 제출, 양수=마감 늦게 제출 |
| `total_click_28` | 28일 동안 온라인 학습 시스템(VLE) 총 클릭 수 |
| `active_days_28` | 28일 중 실제로 접속해서 활동한 날 수 |
| `distinct_resources_28` | 28일 동안 열람한 서로 다른 학습자료 개수 |

## 모델링할 때 절대 피처로 넣으면 안 되는 컬럼 (정답 유출)

`final_result`, `label_churn`, `date_unregistration` — 이 셋은 타겟(`label_churn_28d`)을
계산하는 데 쓰인 재료라서, 예측 모델의 입력값으로 넣으면 안 됨(모델이 답을 미리 보고 맞히는
셈이 됨).
