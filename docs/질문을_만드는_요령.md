# 질문을 만드는 요령 (EDA 인사이트 도출용)

EDA에서 그래프나 통계를 봐도 "그래서 뭘 봐야 하지?"가 막막할 때, 질문을 만드는 4가지 방법을 정리했다.

## 1. 컬럼 의미 + 목표를 결합하기

컬럼 하나를 정하고 "이 컬럼이 크면/작으면, 혹은 이런 값이면 우리가 보려는 목표(예: 이탈)에 영향을 줄까?"를 스스로 물어본다.

- 예: `weight`(평가 비중)를 보면서 → "비중이 큰 평가를 놓치는 것과 작은 평가를 놓치는 것, 학생 입장에서 부담이 다르지 않을까?"

컬럼의 원래 의미(마감일, 비중, 유형 등)를 먼저 떠올리고, 거기에 분석 목표를 갖다 붙이면 질문이 자연스럽게 나온다.

## 2. 그래프에서 눈에 띄는 부분 찾고 "왜?" 붙이기

그래프를 그렸을 때 이상하거나 튀는 부분(치우침, 이상치, 특정 값만 적음/많음)을 찾고, 거기에 "왜 이렇지?"를 붙인다.

- 예: 특정 학기(2013B)만 데이터 개수가 유독 적다 → "왜 이 학기만 적을까?"

좋은 질문의 상당수는 이미 그래프를 눈으로 보고 튀는 부분을 캐치하는 데서 나온다.

## 3. 도메인 지식 대입하기

분석 대상 도메인에서 일반적으로 알려진 패턴을 떠올리고, 그걸 우리 데이터 컬럼에 대입해본다.

- 예: "온라인 강의에서 학생들은 보통 언제 이탈할까?" → 과제 마감 직전, 학기 초반 적응 실패, 시험 부담 등
- 이 일반 지식을 컬럼에 대입: "우리 데이터에도 마감일 직전 이탈이 몰리는 패턴이 있을까?"

## 4. 질문 템플릿 활용하기

컬럼마다 기계적으로 적용할 수 있는 템플릿을 하나 정해두면 빠르다.

> "[속성 X]의 [패턴 Y]가, [분석 목표]와 관련 있을까?"

이 틀에 데이터셋의 컬럼을 하나씩 넣어보면서 연습하면 실전에서 빠르게 질문을 뽑아낼 수 있다.

## 참고: 질문의 범위 구분하기

질문을 만들 때는 "지금 이 파일 하나로 확인 가능한 질문"과 "다른 테이블과 조인해야 확인 가능한 질문"을 구분하는 게 중요하다.

- 단일 파일로 확인 가능 → 바로 코드로 검증
- 조인 필요 → 가설로 남겨두고, 조인 단계에서 검증할 목록에 추가
